As estruturas de dados so formas fundamentais de organizar e armazenar dados em um computador para que possam ser acessados e modificados de forma eficiente. 1.1 Listas

Ns j usamos as Listas anteriormente neste curso. Quando trabalhamos com Vetores e Matrizes, foi dito que elas de fato no existem em Python.Ns utilizamos Listas para a implementao delas. As listas, assim como os vetores, so sequncias ordenadas de itens. Isso significa que cada elemento em uma lista tem uma posio definida (ndice), e essa ordem mantida. Alm de mutveis (permitem alteraes), diferentemente dos vetores em outras linguagens de programao (como C/C++, Java, C# etc), as listas so dinmicas, o que significa que voc pode adicionar ou remover elementos depois que a lista criada.

Uso:

Para criar uma lista, voc usa colchetes [] e separa os elementos por vrgulas.

  • Obter o tamanho da lista: len(lista).

1.1.1 Deque, Uma Outra Forma de Lista

O mdulo collections em Python oferece o deque (double-ended queue), que uma generalizao de pilhas e filas (vista adiante). Este tipo de estrutura de dados suporta adies e remoes eficientes de ambos os lados quando comparados com listas. Em termos de funcionalidade, eles se assemelham a listas, mas oferecem melhor desempenho para operaes de adicionar e remover elementos no incio ou no final da sequncia.

Comparao entre Deque e Lista:

Listas em Python so implementadas em baixo nvel (em C) como vetores dinmicos. Isso significa que so vetores estticos que necessitam ser realocados toda vez que necessrio aumentar o seu tamanho devido a insero de um elemento novo. Alm disto, sempre que um elemento inserido no comeo ou no meio do vetor, todos os valores existentes a partir daquela posio precisam ser copiados para as posies seguintes, consumindo tempo de processamento. Similarmente, de forma inversa, a remoo de um elemento que no o ltimo implica a cpia dos elementos posteriores, cada um, para uma posio imediatamente anterior.

Deques so implementadas em baixo nvel como listas duplamente encadeadas, em que os elementos no esto armazenados em vetores, mas sim em ns conectados por referncias (ponteiros em Linguagem C). Isto significa que a insero de um elemento no incio ou no meio no necessita deslocar os demais. Basta inserir um novo n, conectar este com os imediatamentes anterior e posterior. Na remoo, basta remover o n e conectar os ns vizinhos entre eles.

Uso:

Para usar deque, voc precisa import-lo do mdulo collections.

In[]:
from collections import deque

# Criando um deque
fila_dupla = deque(["elemento1", "elemento2", "elemento3"])
print(f"Deque inicial: {fila_dupla}")

# Adicionando elementos
fila_dupla.append("elemento4")  # Adiciona ao final
fila_dupla.appendleft("elemento0") # Adiciona ao incio
print(f"Deque aps adicionar elementos: {fila_dupla}")

# Removendo elementos
elemento_direita = fila_dupla.pop()   # Remove e retorna o elemento da direita
elemento_esquerda = fila_dupla.popleft() # Remove e retorna o elemento da esquerda
print(f"Deque aps remover elementos: {fila_dupla}, removido da direita: {elemento_direita}, removido da esquerda: {elemento_esquerda}")

Deque inicial: deque(['elemento1', 'elemento2', 'elemento3'])
Deque aps adicionar elementos: deque(['elemento0', 'elemento1', 'elemento2', 'elemento3', 'elemento4'])
Deque aps remover elementos: deque(['elemento1', 'elemento2', 'elemento3']), removido da direita: elemento4, removido da esquerda: elemento0

Quando usar Deque?

O deque particularmente til quando voc precisa de uma estrutura de dados semelhante a uma lista, mas com a necessidade de realizar inseres e excluses eficientes em ambas as extremidades. Isso o torna uma boa escolha para implementar filas (onde voc adiciona em uma extremidade e remove da outra) e pilhas (onde voc adiciona e remove da mesma extremidade).

1.2 Tuplas

As tuplas so semelhantes s listas, mas so imutveis. Uma vez que uma tupla criada, seus elementos no podem ser modificados, adicionados ou removidos. As tuplas so definidas usando parnteses ().

Por que usar Tuplas?

  • Imutabilidade: Garante que os dados permaneam constantes ao longo do programa. Isso pode ser til para representar valores fixos, como coordenadas, dias da semana, etc.
  • Desempenho: Em algumas operaes, as tuplas podem ser ligeiramente mais rpidas que as listas devido sua natureza esttica.
  • Chaves de Dicionrios: Tuplas podem ser usadas como chaves em dicionrios (j que so imutveis), enquanto listas no podem.
  • Retorno Mltiplo de Funes: Funes em Python frequentemente retornam mltiplos valores como uma tupla.

1.3 Dicionrios

Os dicionrios so colees no ordenadas de pares chave-valor. Cada chave em um dicionrio deve ser imutvel e nica, e usada para acessar o valor correspondente. Os dicionrios so mutveis e so definidos usando chaves {}.

Operaes Comuns:

  • Acessar valores: dicionario[chave] (se a chave no existir, levanta um KeyError), .get(chave, valor_padrao) (retorna o valor da chave ou um valor padro se a chave no existir).
  • Adicionar/modificar pares: dicionario[nova_chave] = novo_valor.
  • Remover pares: del dicionario[chave] (levanta KeyError se a chave no existir), .pop(chave, valor_padrao) (remove e retorna o valor da chave; se a chave no existir, retorna o valor padro ou levanta KeyError se nenhum padro for fornecido), .popitem() (remove e retorna um par chave-valor arbitrrio - til para remover itens em ordem LIFO).
  • Verificar a existncia de uma chave: chave in dicionario.
  • Obter todas as chaves: dicionario.keys() (retorna uma view).
  • Obter todos os valores: dicionario.values() (retorna uma view).
  • Obter todos os pares chave-valor: dicionario.items() (retorna uma view de tuplas).
  • Atualizar com outro dicionrio: .update(outro_dicionario) (adiciona ou atualiza pares chave-valor).

1.4 Conjuntos

Conjuntos so colees no ordenadas de elementos nicos. Isso significa que um conjunto no pode conter duplicatas. Os conjuntos so mutveis e so definidos usando chaves {} ou a funo set().

Implementao:

In[]:
# Criando um conjunto
numeros_unicos = {1, 2, 3, 4, 5}
print(f"Conjunto de nmeros nicos: {numeros_unicos}")

frutas_set = set(["ma", "banana", "laranja", "ma"]) # Duplicatas so removidas
print(f"Conjunto de frutas: {frutas_set}")

Operaes Comuns:

Conjuntos suportam operaes matemticas de conjuntos:

  • Adicionar elemento: .add(elemento).
  • Remover elemento: .remove(elemento) (levanta KeyError se o elemento no existir), .discard(elemento) (remove o elemento se ele existir, sem levantar erro).
  • Unio: conjunto1 | conjunto2 ou conjunto1.union(conjunto2).
  • Interseo: conjunto1 & conjunto2 ou conjunto1.intersection(conjunto2).
  • Diferena: conjunto1 - conjunto2 ou conjunto1.difference(conjunto2) (elementos em conjunto1 que no esto em conjunto2).
  • Diferena Simtrica: conjunto1 ^ conjunto2 ou conjunto1.symmetric_difference(conjunto2) (elementos que esto em um dos conjuntos, mas no em ambos).
  • Verificar subconjunto: conjunto1.issubset(conjunto2).
  • Verificar superconjunto: conjunto1.issuperset(conjunto2).
  • Verificar disjuno: conjunto1.isdisjoint(conjunto2) (se no tm elementos em comum).
In[]:
# Adicionando e removendo elementos
numeros_unicos.add(6)
print(f"Conjunto aps adicionar: {numeros_unicos}")

numeros_unicos.remove(3)
print(f"Conjunto aps remover: {numeros_unicos}")

frutas_set.discard("abacaxi") # No existe, mas no levanta erro
print(f"Conjunto de frutas aps discard: {frutas_set}")

# Operaes de conjuntos
conjunto_a = {1, 2, 3, 4}
conjunto_b = {3, 4, 5, 6}

uniao = conjunto_a | conjunto_b
print(f"Unio: {uniao}")

intersecao = conjunto_a & conjunto_b
print(f"Interseo: {intersecao}")

diferenca_ab = conjunto_a - conjunto_b
print(f"Diferena (A - B): {diferenca_ab}")

diferenca_simetrica = conjunto_a ^ conjunto_b
print(f"Diferena Simtrica: {diferenca_simetrica}")

# Verificaes
eh_subconjunto = {1, 2}.issubset(conjunto_a)
print(f"{{1, 2}}  subconjunto de {conjunto_a}? {eh_subconjunto}")

2.1 Pilhas

Uma pilha (stack) uma estrutura de dados que segue o princpio LIFO (Last-In, First-Out), onde o ltimo elemento adicionado o primeiro a ser removido. Podemos implementar uma pilha de forma simples utilizando listas em Python.

Implementao usando Lista:

Para implementar uma pilha com uma lista, as operaes de "empilhar" (adicionar um elemento ao topo) e "desempilhar" (remover o elemento do topo) correspondem, respectivamente, aos mtodos .append() e .pop() da lista. O "topo" da pilha ser sempre o ltimo elemento da lista.

In[]:
# Criando uma pilha vazia
pilha_lista = []

# Empilhando elementos
pilha_lista.append(10)
pilha_lista.append(20)
pilha_lista.append(30)

print(f"Pilha (lista) inicial: {pilha_lista}")

# Verificando se a pilha est vazia
esta_vazia = not pilha_lista
print(f"A pilha est vazia? {esta_vazia}")

# Verificando o topo da pilha
if pilha_lista:
    topo = pilha_lista[-1]
    print(f"Topo da pilha: {topo}")
else:
    print("A pilha est vazia, no h topo.")

# Desempilhando um elemento
elemento_removido = pilha_lista.pop() if pilha_lista else None
print(f"Elemento desempilhado: {elemento_removido}")
print(f"Pilha (lista) aps desempilhar: {pilha_lista}")

# Tamanho da pilha
tamanho = len(pilha_lista)
print(f"Tamanho da pilha: {tamanho}")

# Desempilhando toda a pilha
print("Desempilhando todos os elementos:")
while pilha_lista:
  elemento_removido = pilha_lista.pop()
  print(f" {elemento_removido}")
print(f"Tamanho da pilha: {len(pilha_lista)}")

2.2 Filas

Uma fila (queue) uma estrutura de dados que segue o princpio FIFO (First-In, First-Out), onde o primeiro elemento adicionado o primeiro a ser removido.

Fonte da imagem: Internet.

Implementao usando Lista (Ineficiente):

Podemos tentar implementar uma fila usando uma lista, onde a adio de elementos ocorre no final (append()) e a remoo ocorre no incio. No entanto, remover o primeiro elemento de uma lista (pop(0)) uma operao ineficiente, pois todos os outros elementos precisam ser deslocados, resultando em uma complexidade de tempo O(n), onde n o nmero de elementos na lista.

In[]:
# Criando uma fila vazia (usando lista - INEFICIENTE para remoo)
fila_lista = []

# Enfileirando elementos
fila_lista.append("primeiro")
fila_lista.append("segundo")
fila_lista.append("terceiro")

print(f"Fila (lista) inicial: {fila_lista}")

# Verificando se a fila est vazia
esta_vazia_fila_lista = not fila_lista
print(f"A fila (lista) est vazia? {esta_vazia_fila_lista}")

# Desenfileirando um elemento (ineficiente)
elemento_removido_fila_lista = fila_lista.pop(0) if fila_lista else None
print(f"Elemento desenfileirado (lista): {elemento_removido_fila_lista}")
print(f"Fila (lista) aps desenfileirar: {fila_lista}")

# Verificando o primeiro elemento da fila
if fila_lista:
    primeiro_elemento_fila_lista = fila_lista[0]
    print(f"Primeiro elemento da fila (lista): {primeiro_elemento_fila_lista}")
else:
    print("A fila (lista) est vazia.")

# Tamanho da fila
tamanho_fila_lista = len(fila_lista)
print(f"Tamanho da fila (lista): {tamanho_fila_lista}")

# desenfileirando todos
print("\nDesenfileirando todos:")
while fila_lista:
  e = fila_lista.pop(0)
  print(e)
print(f"Tamanho da fila (lista): {len(fila_lista)}")

Implementao usando deque (Eficiente):

O deque do mdulo collections a estrutura ideal para implementar filas de forma eficiente. Podemos usar append() para adicionar elementos ao final (enfileirar) e popleft() para remover elementos do incio (desenfileirar), ambas com complexidade de tempo O(1).

In[]:
from collections import deque

# Criando uma fila vazia usando deque
fila_deque = deque()

# Enfileirando elementos
fila_deque.append("primeiro")
